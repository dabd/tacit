# Foundations

Tacit uses one preservation model for three operations. Prose is the
conservative base. Compress selects content. Laconic changes register.

## Shared preservation contract

Unless the user explicitly requests a content change, preserve:

- facts, intent, decisions, requirements, proposals, and modality;
- uncertainty, assumptions, confidence, and attribution;
- numbers, identifiers, links, code symbols, and technical terms;
- reasoning, evidence, constraints, risks, trade-offs, and actions;
- owners and deadlines.

Do not invent context, increase certainty, alter attribution, or assign
unsupported blame.

## Macrostructure

Macrostructure means headings, section order, and the document's overall
organisation. Prose preserves it. Laconic changes register within it. Compress
may reconstruct it when selecting and arranging the content required for the
document's purpose.

## Prose

Prose improves clarity without changing information or macrostructure. It may
improve agency, verb choice, sentence structure, terminology, and local
wordiness. It removes only wording that contributes no information. Clean prose
stays unchanged.

## Compress

Compress removes optional content only after identifying the information
required for the document's purpose. It builds one semantic inventory,
reconstructs once from the retained information, and checks the result against
both the source and the inventory. It does not impose a terse register.

## Laconic

Laconic changes retained information to a terse, direct register. It may remove
zero-information wording, but it does not decide which unique information is
optional.

## Composition

When both modifiers are requested, apply Compress first and Laconic second.
Build the semantic inventory before compression, apply Laconic only to the
compressed candidate, then check the final result against the source and the
retained inventory.

## Output

Return paste-ready text by default. Add critique, alternatives, annotations, or
an explanation only when the user asks for them.
