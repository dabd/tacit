# Laconic

Change the register, not the information. Do not remove unique information.

Follow the [shared preservation contract](../../FOUNDATIONS.md#shared-preservation-contract),
the [Laconic operation](../../FOUNDATIONS.md#laconic), and the
[composition rule](../../FOUNDATIONS.md#composition).

## Register

- Preserve facts, intent, proposals, modality, reasoning, evidence,
  uncertainty, attribution, numbers, identifiers, links, code symbols,
  technical terms, constraints, risks, actions, owners, deadlines, and stance.
- Put one main thought in each sentence when splitting does not obscure a
  logical relation.
- Prefer a period when a comma joins independent thoughts.
- Remove a transition only when sentence order still expresses the relation.
- Keep required contrasts, causes, conditions, sequences, qualifications, and
  emphasis.
- Remove zero-information wording and semantic duplication. Do not decide that
  unique information is optional.
- Prefer literal verbs and ordinary technical wording.
- Avoid a long run of equal-length sentences when small variation adds no
  content.

Do not apply this register automatically.

## Composition

When both modifiers are requested, apply Compress first and Laconic second.
Apply this register only to the information retained by Compress.

## Output

Return only the paste-ready revised text unless the user asks for an
explanation. If the text already uses the requested laconic register, return it
unchanged.
